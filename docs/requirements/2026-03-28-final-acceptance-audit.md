# 2026-03-28 收官验收需求

## 目标

对当前只读额度查看器做一轮收官验收，回答：

1. 现在是否已经达到“整体正常可用”
2. 是否仍存在 bug、错误、冗余、残留或明显体验问题
3. 哪些问题属于必须继续修，哪些只是低优先级打磨

## 范围

- 当前 `~/.codex` 账号额度读取
- `cc-switch` 普通登录账号只读额度读取
- 菜单栏标题、菜单文案、设置文案
- 测试、构建、打包验证链
- 仓内残留代码、残留文档、命名与治理产物

## 约束

- 保持产品定位为只读额度查看器
- 不恢复账号切换能力
- 不覆盖当前 `auth.json` 和 `config.toml`
- 不回滚无关 worktree 改动

## 验收标准

- `swift test`、`swift build` 可用
- 关键功能无高优先级错误态误判或主链路阻塞
- 菜单文案简洁，不再明显误导为切换器
- 不存在明显会把旧 bug 带回来的死入口
- 结论必须区分：通过项、剩余风险、建议优化项

## 非目标

- 新增产品功能
- 做签名、公证、CI 基建扩展
